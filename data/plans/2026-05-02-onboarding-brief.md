# Дизайн-бриф: Первый запуск (Onboarding) + Пустые состояния

## 1. Описание фичи

Первый запуск для новых пользователей платформы. После регистрации пользователь выбирает роль (фигурист / тренер / хореограф), отвечает на короткий вопрос (откуда узнали о нас), затем проходит ролевой поток возможностей за 60 секунд. Поток всегда можно пропустить. После — контекстуальное пустое состояние, подталкивающее к первому значимому действию.

## 2. Главное действие пользователя

**Фигурист:** загрузить первое тренировочное видео и получить биомеханический анализ.
**Тренер:** пригласить первого ученика или просмотреть демо-данные прогресса.
**Хореограф:** (future) создать первую программу или ознакомиться с инструментами планирования.

## 3. Направление дизайна

**Цветовая стратегия:** Сдержанная (Restrained). Ice Deep ≤ 10%.

**Сцена:** Фигурист на скамейке у борта, телефон в руке, 5 минут после тренировки. Уставший, но собранный. Интерфейс — спокойный, сканируемый, быстрый.

**Референсы:** Strava first-run, Apple Fitness+ onboarding, Notion empty states.

## 4. Объём

- **Качество:** High-fi, production-ready
- **Охват:** Ролевой выбор → 3-слайдовый тур → пустой dashboard + reusable empty-state компоненты
- **Интерактивность:** Функциональный прототип с real state transitions
- **Цель:** Production quality

## 5. Стратегия раскладки

### Шаг 1: Выбор роли + откуда узнали
Простая форма: "Кто вы?" (3 radio-кнопки: Фигурист / Тренер / Хореограф) + опционально "Как вы узнали о нас?" (select). Одна кнопка "Продолжить".

### Шаг 2: Тур по возможностям (3 слайда, ролевой)

**Фигурист:**
1. "Загрузите тренировочное видео" → "Запишите или загрузите любой прыжок или элемент."
2. "Мгновенная биомеханика" → "Поза, метрики и техника за секунды."
3. "Отслеживайте прогресс" → "Следите за ростом и ставьте личные рекорды."

**Тренер:**
1. "Пригласите учеников" → "Отправьте ссылку или найдите по имени."
2. "Следите за прогрессом" → "Высота прыжков, ротация, качество приземления."
3. "Давайте обратную связь" → "Комментируйте видео, даже на расстоянии."

**Хореограф:**
1. "Загрузите музыку" → "Автоматический анализ BPM и структуры."
2. "Планируйте программу" → "Расставляйте элементы по таймингу."
3. "Визуализируйте раскладку" → "2D-схема катка с траекториями."

### Шаг 3: Пустой dashboard
Никаких фейковых данных. Одна заметная карточка CTA по центру. Окружающие области — подсказки-структуры.

### Пустые состояния (reusable)
- **Лента (нет сессий):** ice-иллюстрация + "Загрузите первое видео" CTA + "Связаться с тренером" (если нет связи).
- **Прогресс (нет данных):** "Проанализируйте видео, чтобы увидеть тренды."
- **Связи (нет тренера):** "Пригласите тренера или найдите его."
- **Ученики (для тренера, нет учеников):** "Пригласите первого ученика."

## 6. Ключевые состояния

| Состояние | Что видит | Что чувствует |
|---|---|---|
| **Первый визит** | Ролевой выбор + тур | Любопытство, направленность |
| **Пропущенный** | Пустой dashboard с CTA | Контроль, нет вины |
| **Загрузка** | Индикатор, затем обработка | Доверие |
| **После анализа** | Первые метрики | Успех, подтверждение ценности |
| **Ошибка** | Inline retry, без модала | Разочарование сдержано |
| **Приглашённый ученик** | Dashboard с пре-установленной связью с тренером | Готовность начать |

## 7. Модель взаимодействия

- **Регистрация → Роль → Тур:** Последовательно, с кнопкой "Пропустить" всегда видной.
- **Тур → Dashboard:** После 3-го слайда или пропуска. Fade transition.
- **Приглашённые пользователи:** Skip "связаться с тренером", связь pre-established. Флаг в URL-параметре или invite token.
- **Пропуск:** Флаг в localStorage. Не показывать снова до сброса в настройках.
- **Reduced motion:** Мгновенные fades, никакого sliding.

## 8. Требования к тексту

**Тон:** Прямой, уверенный, без лишнего. Без тире.

- Заголовок: "Ваш AI-тренер по фигурному катанию"
- Подзаголовок: "Загрузите видео. Получите биомеханику. Отслеживайте прогресс."
- Слайды: см. раздел 5.
- Пустая лента: "Пока нет сессий" / "Загрузите первое видео, чтобы получить биомеханический анализ."
- Ссылка пропуска: "Пропустить"
- Вопрос "Кто вы?": "Фигурист", "Тренер", "Хореограф"
- Вопрос "Откуда узнали?": "От друга", "В соцсетях", "На соревнованиях", "От тренера", "Другое"

## 10. Решённые вопросы

1. **Ролевой onboarding:** Да, 3 роли — Фигурист, Тренер, Хореограф. Разные flows.
2. **Хореографы:** Включены как future-ready. Сейчас placeholder flow.
3. **Гибкость ролей:** Система bidirectional. Пользователь может менять роли. Роль при регистрации — начальная точка, не пожизненная.
4. **Приглашённые ученики:** Skip "связаться с тренером", связь pre-established.
5. **H-Sense:** Не упоминать в onboarding. Упомянуть на лендинге позже.
6. **Клубы/школы:** Future concept, не в onboarding сейчас.
